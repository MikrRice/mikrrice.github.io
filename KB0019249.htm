What's the best plan to upgrade to 12.0+ & move to a new sql 2012 or 2014 server?
KB0019249
Product this article applies to: Prophet 21
Published to customer portal:Yes

Overview

This solution details recommended step for moving to any version of 12.0 or higher and upgrading to a new SQL server.  

Suggested Conversion Steps

These are the general steps we recommend.  These steps are by no means required nor are they exhaustive.  You should perform testing of these steps to understand how they work before implementing them.  Every customer's situation is different and you may have specific needs for your business.  If you have any specific questions, log a case with P21 support before moving forward on any plan of action.

StepAction
1

Review KB0018385 for general things to do when moving from one SQL server to another.  Check these first to understand what you'll need to do before going live.

2

Review KB0019213 and note the versions of SQL server that are and are not supported for your version.

3

Record and review any SQL jobs, maintenance plans or any other functions you may have had on your old SQL server.  These are things you should review on your new server after moving your DB.

4If you are on SQL 2008 or higher and your server is up to hardware specifications, we recommend you upgrade your database in place on your existing server first and test to make sure everything works.  You can proceed after you feel the current environment is stable and performs appropriately.  You do not have to do this but there are fewer things to check at this step if something goes wrong.
5If you meet any of the following scenarios:
  • On SQL 2000
  • On SQL 2005 and upgrading to 12.17 or higher
  • You current hardware may not meet the current hardware specifications after upgrading to the current V12 version.

You will not be able to perform step 4, which means you'll need to upgrade to V12 and SQL server versions at the same time.  It would be best at this point to backup your current DB on the existing server and restore to the new server, and then perform the update.

6Most customers simply purchase new hardware and move between servers.  Very few customers upgrade the SQL server in place, but you can do this if needed.  You should backup your entire SQL server before doing this, in case anything goes wrong.  It's not recommended because if something goes wrong while moving to a new server, you can just go back to your old hardware.
7If you used step 5 here, at this step, reapply the SQL portion of the update again.  This will make sure any switches are necessary changed by the update after moving to a new version of SQL server.
8

Perform steps you found necessary in step 1 in KB0018385.

9

Perform any and all appropriate tests to make sure the server meets your business needs before going live.
